3.1.1  框架定位说明

jahhan采用微核心+插件机制，是数据治理框架，可与dubbo，spring boot等服务治理框架集成。

common-plugin-util为核心插件工具包，可依赖此包扩展其他插件逻辑。

frameworkx为jahhan内置的服务治理框架，为dubbox修改版本，修改了dubbo的插件机制，改用jahhan的插件机制。

3.3    规范说明

6.3   

3.3.1  建表规范

1         表命名规范：

表以 t_开头.配置表以 config 结尾. 信息表以 info 结尾. 关系表以 relation 结尾. 流水表以 flow 结尾. 状态表以 status 结尾. 记录表以 record 结尾. 临时表以 temp 结尾. 日志表以 log 结尾.

2         表及字段comment的三段定义

$属性多用于表comment，@属性多用于字段，通常情况下可不指定后两段定义。

代码生成属性说明：

属性名称

属性说明

localCache

是否使用本地缓存

redisCache

是否使用redis缓存

methodExtention

是否生成扩展的数据库查询方法

 

3         必须创建并且只能创建一个字段做为主键

4         字符串字段尽量指定字符串长度，尽量指定是否非空（影响生成代码）

5         不要使用外键（影响性能及解耦性）

6         必须包含modify_timestamp字段，为非空bigint类型

3.3.2  工程结构及类命名规范

工程分三类：接口工程，服务工程及sdk工程，接口工程以intf-开头，服务工程以service-开头，sdk工程以sdk-开头。

 

接口工程结构：

包命名以[net.jahhan|服务名路径]开头

底下需包含intf，vo，constant3个子目录

intf放接口类，vo放vo类，constant放常量类

接口类需以Intf结尾，vo类分入参和出参，入参以IVO结尾，出参以OVO结尾

 

服务工程结构：分3层：controller层，service层，dao层

controller层，service层包命名以[net.jahhan|服务名路径].[ controller|service]开头

dao层为代码自动生成目录结构及类文件，在生成的基础上修改。

controller层的类文件以Controller结尾，需继承Intf结尾的接口。controller用作业务校验。

service层的类文件以Service结尾。service层用作业务逻辑。

服务工程根目录下需包含doc文件夹，里面至少要包含设计文档和powerdesigner表结构设计文档。

5.1.1  说明

框架允许在公司名称+init.initer|module包底下添加启动器。

module模块需继承AbstractModule类，initer需实现BootstrapInit接口，两种初始化器都需有@InitAnnocation注解。

@InitAnnocation注解中，isLazy参数为是否懒加载，默认为false，如果为true，将在同类启动器都完成后执行;initSequence为启动顺序，默认为9999，isLazy为false是生效，启动顺序不能重复。onlyWeb为true时，非web工程启动时不加载，默认为false，web工程不管true和false都加载。业务层启动顺序后两位不能都为0。

5.1.2  实例

Module实例：

@InitAnnocation(isLazy = false, initSequence = 2010)

public class BizModule extends AbstractModule {

 

    private static Logger logger = LoggerFactory.getLogger(BizModule.class);

 

    @Override

    protected void configure() {

    bind(ISmsAgent.class).annotatedWith(Names.named(SmsAgentType.MAIN.name())).to(RuanweiSms.class);

    bind(ISmsAgent.class).annotatedWith(Names.named(SmsAgentType.EXTRA.name())).to(QixinggangSms.class);

    }

 

    @Singleton

    @Named("PushExecutor")

    @Provides

    ScheduledExecutorService providePushExecutor() {

        final ScheduledExecutorService pushExecutor = Executors.newScheduledThreadPool(1);

 

        Runtime.getRuntime().addShutdownHook(new Thread() {

            @Override

            public void run() {

                try {

                   pushExecutor.shutdown();

                   if (!pushExecutor.awaitTermination(5, TimeUnit.SECONDS)) {

                       pushExecutor.shutdownNow();

                   }

                } catch (Exception e) {

                   logger.error("error when shutdown PushExecutor", e);

                   Thread.currentThread().interrupt();

                }

            }

        });

 

        return pushExecutor;

    }

}

@Extension("fixed")

@Singleton

public class FixedThreadPool implements ThreadPool {

 

    public Executor getExecutor(URL url) {

        String name = url.getParameter(Constants.THREAD_NAME_KEY, Constants.DEFAULT_THREAD_NAME);

        int threads = url.getParameter(Constants.THREADS_KEY, Constants.DEFAULT_THREADS);

        int queues = url.getParameter(Constants.QUEUES_KEY, Constants.DEFAULT_QUEUES);

        return new ThreadPoolExecutor(threads, threads, 0, TimeUnit.MILLISECONDS,

             queues == 0 ? new SynchronousQueue<Runnable>() :

                (queues < 0 ? new LinkedBlockingQueue<Runnable>()

                        : new LinkedBlockingQueue<Runnable>(queues)),

             new NamedThreadFactory(name, true), new AbortPolicyWithReport(name, url));

    }

 

}

5.3    接口开发

6.3   

5.3.1  接口注解说明

接口需添加rest，swagger，hibernate validation 注解。

@Path("third")指定rest请求路径

@Consumes({ ContentType.APPLICATION_JSON_UTF_8 })接受的MIME类型

@Produces({ ContentType.APPLICATION_JSON_UTF_8 })输出的MIME类型

@Api("第三方服务接口")接口分类的描述

@GET接受的请求类型（@POST，@PUT，@DELETE）

@ApiOperation(value = "鉴权")接口描述

@QueryParam(value = "token")入参方式（@QueryParam为？token=xxx，@PathParam为token/xxx/）

@NotNull(message = "token不能为空")为hibernate校验类型

VO分为两种，入参为IVO，出参为OVO

@Data为lombok注解，可免写getset方法

@NotNull(message = "service不能为空")为hibernate校验类型

@ApiModelProperty(value = "服务名称")为字段说明

 

接口中的方法可添加注解@DubboMethod来控制dubbo的method具体配置，包括异步请求等，具体使用方式可参照<dubbo:method/>

接口中的方法请求参数字段可添加注解@DubboArgument使用方式可参照<dubbo:argument/>

 

接口类可添加@DubboInterface注解来做指定服务器的推送。clusterType = RPCTypeEnum.DIRECT（默认为CLOUD）, target = "xxxservice"（指定推送的服务器集群类型），若指定服务器推送，则接口方法的第一个参数应为string类型，为指定推送服务的唯一id。服务节点的唯一id可根据BaseContext.CTX.getNodeId()获得。

5.4    controller层开发

6.4   

5.4.1  说明及实例

controller类需实现intf接口，在实现中做业务校验，预定义值，返回值修改。类需要添加@Controller及@Singleton注解。

 

@Controller

@Singleton

public class ServiceController implements ServiceIntf {

    @Inject

    private ServiceService serviceService;

    @Inject

    private TServiceDao tServiceDao;

 

    private Redis redis = RedisFactory.getRedis(RedisConstants.COMMON, null);

 

    @DBConnections({ @DBConnect(dataSource = "bpm2",value=DBConnectStrategy.ORIGINAL) })

    @GlobalSyncTransaction

    @Override

    @Cache(blockTime = 10)

    public ServiceRegisteOVO register(ServiceRegisteIVO serviceRegisteIVO) {

        String serviceType = serviceRegisteIVO.getServiceType();

        ServiceTypeEnum serviceTypeEnum = ServiceTypeEnum.valueOf(serviceType);

        String serviceCode = serviceRegisteIVO.getServiceCode();

        if (serviceTypeEnum.equals(ServiceTypeEnum.inner) && null == serviceCode) {

            JahhanException.throwException(HttpServletResponse.SC_BAD_REQUEST, SystemErrorCode.VALIATION_EXCEPTION,

                   "serviceType 为inner时serviceCode 不能为空！");

        } else if (serviceTypeEnum.equals(ServiceTypeEnum.third)) {

            serviceCode = UUID.randomUUID().toString().replaceAll("-", "");

        }

        return serviceService.register(serviceRegisteIVO);

    }

}

实现方法可定义注解：

@DBConnections({ @DBConnect(dataSource = "bpm2",value=DBConnectStrategy.ORIGINAL) })为指定数据源及对应的数据连接选择策略及初始连接等级，默认选择策略为升级策略，连接等级为0

连接等级：0无连接，1读连接，2写连接，3批写连接

升级策略：当数据库操作大于当前连接等级时，则升级，若小于当前连接等级时，则使用当前连接，适用于事务一致性要求操作。

原等级策略：不够当前连接等级是什么，使用与操作同级的数据库连接，适用于无事务一致性要求，该策略性能较好。

若有写请求，在第一次写操作的时候开启事务，接口结束时自动关闭。

因为连接策略特性，开发时尽量把等级更高的操作写在后面，这样高级连接占时比较短，可提升性能。

 

@GlobalSyncTransaction为全局事务入口注解

@Cache为接口缓存注解，fastBackFail为是否快速返回失败（true：接口调用阻塞期间返回失败，false：接口调用阻塞期间返回缓存结果），blockTime为阻塞时长（秒）

@NoneToken免密接口注解（无需加密）

@FirstToken首登接口注解（需加密，用于页面端第一次）

@RequestMode为请求模式，值value分别指向普通请求和登陆请求（需加密，用于app端）

@ServiceAuthentication为服务器鉴权注解，可用AuthenticationVariable.getAuthenticationVariable().getService()获取服务器信息

@UserAuthentication为用户鉴权注解

AuthenticationVariable.getAuthenticationVariable().getUser()获取用户信息

5.7.1  异常类JahhanException

业务异常需继承或者使用JahhanException：

JahhanException.throwException(HttpServletResponse.SC_BAD_REQUEST, JahhanErrorCode.VALIATION_EXCEPTION, "错误信息！");

抛出的异常需指定http状态码，异常错误码及错误信息

http状态码说明：

2xx （成功）

表示成功处理了请求的状态代码。

 

代码 说明

200 （成功） 服务器已成功处理了请求。通常，这表示服务器提供了请求的网页。

 

3xx （重定向）

表示要完成请求，需要进一步操作。 通常，这些状态代码用来重定向。

 

代码 说明

302 （临时移动） 服务器目前从不同位置的网页响应请求，但请求者应继续使用原有位置来进行以后的请求。

304 （未修改） 自从上次请求后，请求的网页未修改过。服务器返回此响应时，不会返回网页内容。

 

4xx（请求错误）

这些状态代码表示请求可能出错，妨碍了服务器的处理。

 

代码 说明

400 （错误请求） 服务器不理解请求的语法。

401 （未授权） 请求要求身份验证。 对于需要登录的网页，服务器可能返回此响应。

403 （禁止） 服务器拒绝请求。

405 （方法禁用） 禁用请求中指定的方法。

 

5xx（服务器错误）

这些状态代码表示服务器在尝试处理请求时发生内部错误。 这些错误可能是服务器本身的错误，而不是请求出错。

 

代码 说明

500 （服务器内部错误） 服务器遇到错误，无法完成请求。

注：400指向客户端错误，500指向服务端错误
